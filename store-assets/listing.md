## English

### Summary

Beautify Mermaid & PlantUML diagrams in-place on GitHub, GitLab, Jira, Confluence, dev.to, and more.

**Character count:** 93

### Detailed description

Beauty Diagram re-renders Mermaid and PlantUML diagrams directly on the pages you already visit — GitHub pull requests, GitLab merge requests, Jira and Confluence (*.atlassian.net), dev.to, Stack Overflow, Linear, Hashnode, ChatGPT, Claude, Gemini, Perplexity, and www.beauty-diagram.com. No copy-pasting, no separate tab: the improved diagram replaces the original on the page, and the original source stays available.

On those built-in sites the extension runs automatically. For any other site — say, an internal wiki or a custom documentation portal — you can enable it for that site from the popup with a single click. Raw code blocks (fenced ` ```mermaid ``` ` or ` ```plantuml ``` ` that have not yet been rendered) are never auto-processed: they show a small **◇ Render** button so you decide which diagrams get beautified.

The rendering is done by the Beauty Diagram API (api.beauty-diagram.com), the same service behind the VS Code and Obsidian Beauty Diagram plugins. Only the diagram source text is sent — no page content, no browsing history, no personal data. An optional Pro API key removes the render watermark; it is stored only in your browser's local storage, never synced or logged.

**Features**

- In-place beautification — diagrams update on the page, no new tab needed
- **◇ Render** opt-in button for raw code blocks (never auto-rendered)
- Automatic on 13 built-in sites; per-site toggle for any other site via the popup
- Supports both Mermaid and PlantUML
- Theme selector and max-width control
- Original diagram source is always kept and accessible
- Optional Pro API key for watermark-free output
- No account required, no analytics, no ads

- 13 個內建網站自動執行；透過彈出視窗對任何其他網站開啟個別切換
